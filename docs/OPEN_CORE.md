# SecuGent Open Core Model

SecuGent uses an open-core model with a hard dependency boundary.

## License Tiers

| Tier | License | Scope |
| --- | --- | --- |
| Core | Apache-2.0 | Policy engine, Rule of Two, audit/hash-chain primitives, local SDK and CLI trust primitives. |
| Enterprise | LicenseRef-SecuGent-Enterprise | Commercial add-ons such as multi-tenant administration, external KMS providers, console/compliance surfaces, quota enforcement, and SSO/RBAC integrations. |

## Package Boundary

Core modules must not import Enterprise modules. The allowed dependency direction
is Enterprise -> Core only.

## CI Gate

`tests/unit/test_open_core_boundary.py` enforces the boundary with AST checks and
SPDX header checks. The gate is deliberately static so Core can be tested without
installing Enterprise extras.

## License Text Handling

`LICENSE` contains the Apache-2.0 Core license. `LICENSE.enterprise` contains the
project-specific Enterprise license notice and SPDX reference. Full standard
license text, if required for a release artifact, should be placed manually from
the official source rather than generated by automation agents.

## Adoption Telemetry (opt-in, default-off, local-first)

`secugent/observability/telemetry.py` provides an **opt-in** adoption-metrics
collector (`TelemetryCollector`) for the 6-month adoption KPI and OEM
conversations. It is privacy-respecting by construction:

- **Default-off.** Telemetry is a complete no-op until an operator explicitly
  sets `SECUGENT_TELEMETRY_OPTIN=1`
  (`secugent.core.settings.TelemetrySettings`). While off, nothing is created,
  buffered, or sent. No Prometheus collector is registered on any global
  registry, so the metric never appears on `/metrics` — not even as HELP/TYPE
  metadata — regardless of opt-in.
- **No PII / no policy content (enforced, not just documented).**
  `record_feature(name)` increments an in-memory count for a feature *name*
  only, and the name is **validated against `[a-z0-9._]{1,64}`** and rejected
  otherwise. A caller cannot smuggle an email, RRN, path, tenant id, or any free
  text / non-ascii string through the name (deny-by-default, §A-2.2). Distinct
  names are cardinality-bounded (overflow bucket beyond `max_features`) so memory
  and label cardinality stay bounded. `snapshot()` returns only `{feature: count}`
  aggregates.
- **Pseudonymous instance id (not anonymous).** `instance_hash()` is a **keyed
  HMAC-SHA256** over an opaque install/host value, keyed by a *per-install secret*
  (random by default, persistable, never emitted). Same id + same secret ⇒ same
  digest; different secret ⇒ different digest. The raw value is never put in the
  flushed payload. This is a stable per-install *pseudonym* — reversal is hard
  only for an adversary who lacks the per-install secret. We do **not** claim a
  bare SHA-256 of a low-entropy hostname is irreversible (it would be
  dictionary-recoverable under a public salt); the secret keying is what defeats
  that attack.
- **Local-first, fail-soft, in-memory/sink-only.** The optional `TelemetrySink`
  flushes the aggregate to a local buffer/file — there is **no mandatory external
  SaaS egress** (§A-2.6 closed-network first) and **no Prometheus export**. A sink
  error (e.g. disk full) is swallowed at the telemetry boundary so the
  application is never affected.
- **Fully separable.** This module imports neither the audit hash-chain nor
  REGULATIONS; telemetry is not entangled with the six Grafana dashboard metrics
  (which live separately in `metrics.py`).
